
**Assessment:**
- Cloudflare is actively caching all page types (`cf-cache-status: HIT`).
- Brotli compression enabled.
- `stale-while-revalidate=86400` is excellent for crawlers.

The Redis cache middleware at `discovery.py:263` sets `max-age=300`, but the actual response shows 14400. This suggests Cloudflare may be rewriting the `max-age` value. Not necessarily a problem but worth understanding.
- **No issues blocking Apple:** All cache headers are well-formed and cache-friendly.

### Apple News publisher eligibility

Apple News publisher requirements (per Apple's public documentation):
1. Must be a recognized news publisher or magazine
2. Must have an editorial team
3. Content must be editorial/journalistic in nature
4. Must apply through Apple News Publisher portal

**Assessment: Nerq does not qualify.** Nerq is a data/trust intelligence platform, not a news publisher. It has no editorial team, no journalistic content, and no bylined articles. The RSS feeds contain automated trust score updates, not editorial content.

**Recommendation:** Do not pursue Apple News. The existing Atom feed at `/feed/recent` is valuable for other purposes (feed readers, AI crawlers) but is not Apple News compatible by content type.

**Recommended: Render-on-demand with Cloudflare caching.**

1. Add a new route `/og/{slug}.png` that generates a 1200x630 PNG card containing:
   - Entity name (large, bold)
   - Trust score (large number with grade badge)
   - Category/registry label
   - Nerq branding

2. Use Python's `Pillow` library for generation (already available or easily installable).

3. Set `Cache-Control: public, max-age=604800, s-maxage=604800, immutable` on the response — OG images change rarely.

5. Fallback: if Pillow is too heavy for Mac Studio's memory pressure, use SVG-to-PNG via `cairosvg` or serve SVG directly (not all consumers support SVG OG images — PNG is safer).

## Risks and open questions

1. **TTFB for external clients unknown.** Mac Studio TTFB is 28-37ms through Cloudflare, but external clients (especially in Australia, Asia, South America) may see 200-500ms depending on Cloudflare edge proximity. Phase 0 cloud migration (Hetzner) may change this significantly. Cannot determine external TTFB without Mac Mini or external vantage point testing.

2. **OG image generation at scale may exceed Mac Studio capacity.** Mac Studio runs at 95% RAM constantly (per CLAUDE.md). Pillow image generation for 5M+ entities, even on-demand, creates memory pressure. Mitigation: aggressive caching + Cloudflare. But first-request storms (e.g., if Apple re-crawls thousands of pages simultaneously) could spike load. Need to measure.

3. **Schema.org changes risk breaking sacred elements.** The SpeakableSpecification (line 7988), pplx-verdict (template line 74), and ai-summary (template line 92) are sacred per ADR-003. Any Schema.org change must pass golden file tests. The proposed changes are additive (new fields in existing JSON-LD) and should not affect sacred elements, but testing is mandatory before deploy.

4. **Apple News is not worth pursuing** given editorial team requirement. Documented above.

5. **Broken og:image on homepage.** `homepage_i18n.py:3205` references `https://nerq.ai/static/og-nerq.png` which does not exist on disk. This is a pre-existing bug unrelated to the Apple sprint but should be fixed in the same deploy.

6. **`max-age=14400` discrepancy.** The live response shows `max-age=14400` (4h) but code sets `max-age=300` (5min). Either Cloudflare is rewriting this or there's an intermediary setting. Not blocking but should be investigated.

7. **Apple Intelligence citation patterns are opaque.** There is no public documentation on how Apple Intelligence selects sources for Siri, Safari Suggestions, or Spotlight. All optimizations are best-effort based on Apple's published Applebot documentation and general SEO best practices.

8. **ZARQ (zarq.ai) robots.txt lacks Applebot.** If Apple begins crawling zarq.ai, it would only see the generic `User-agent: *` rule. Consider adding Applebot directives to the ZARQ section at `zarq_machine_discovery.py:30-104`.

9. **RSS format is Atom, not RSS 2.0 for entity feeds.** Apple News requires RSS 2.0. The `/feed/recent` route produces Atom XML. While we're not pursuing Apple News, if that changes, the feed format would need conversion.

10. **No apple-touch-icon files exist.** Need to generate 4 PNG files (120, 152, 167, 180px) from existing logo before meta tags can reference them.
