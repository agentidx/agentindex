## SYSTEMSTABILISERING

### Problem som löstes

1. **PostgreSQL deadlock (8+ timmar):** En `ALTER TABLE agents ADD COLUMN trust_grade` tog exklusivt lås på agents-tabellen (4.9M rader, 15GB). Alla efterföljande queries stackade upp bakom låset. Lösning: terminerade blockerande query + alla väntande connections.

2. **Parser nere sedan fredag:** Processen var suspenderad (state T). Dödade och lät LaunchAgent starta om.

3. **Duplicerad orchestrator:** Två instanser körde samtidigt (en LaunchAgent, en manuell). Dödade den manuella.

4. **Orchestrator avregistrerad:** LaunchAgent hade blivit avregistrerad efter kill. Krävde `launchctl load` (inte bara `kickstart`).

### Skydd som infördes

**PostgreSQL-nivå (permanenta, databasinställningar):**
```
idle_in_transaction_session_timeout = 300s   (5 min)
statement_timeout = 120s                      (2 min)
```
Förhindrar att en enda query kan låsa systemet i timmar. Om man behöver köra lång DDL: `SET statement_timeout = '30min';` i den sessionen.

**Buzz — Automatisk systemövervakning + självläkning:**
- `~/agentindex/system_healthcheck.py` — körs var 5:e minut via cron, skriver status till `logs/healthcheck.db`
- `~/agentindex/system_autoheal.py` — körs var 5:e minut via cron, fixar kända problem automatiskt

Buzz kan självläka:
- PostgreSQL stuck queries >5 min → dödar dem
- Idle-in-transaction >3 min → dödar dem
- Lock storms (>3 waiters) → dödar alla
- API nere → kill + LaunchAgent restart
- Parser suspenderad/saknas → kill/kickstart/load
- Orchestrator saknas → kickstart, fallback till load
- Duplicerade orchestrators → dödar extras
- Dashboard/MCP saknas → kickstart/load
- Redis nere → brew services restart
- Hung cron-jobb >60 min → kill
- Högt connection count >40 → rensa idle

**Dashboard-widget:** "Buzz — System Monitor" visar status (HEALTHY/WARNING/ERROR), trendprickar, och auto-heal-logg direkt i `nerq.ai/admin/dashboard`.
